SEP11- Project 2 : loops, object and APIS

Throughout my project 2, I demonstrate an understanding of loops, objects, and API. I got to learn how to pull data using API and contribute it to my project. In this project, I created a program with my groupmates Mera and Amia that allows user to type their input and give them the result of their input. In our program, you'll be asked to input what type of store you would like to visit in Filton Street Mall, and we will list out all the stores that fall into the user's input "store type" category. The project helps to filter out the API data to the user's input.

We faced a few challenges when we are in the process of creating this project. At first, we have a little bit confuse about API and how it works. Therefore, we take a bit while to figure out how to make the project work. We ended up resolving that problem when we start playing around. Our second challenge is how to debug the loops formatting through the given API. Although it is simple debugging, we are not satisfied with the way the data loops through. Initially, we only have a "for loop" that isn't specific enough to perform our result ``` for(var i = 0; i < data.length; i++){ ```, so we ended up having every store names and type listed out for the user instead of the type of store user inputted. It makes us use the "if loop" to helps us debug the loops more specifically. We use ``` if (data[i].type == usersStore) {``` to states that if the user's input store type (Ex: Food) match the store type in the data (Ex: Food), we would list out all the Food stores available in the data for the user to see. Our last challenge that our program will have an "UNDEFINED" appear next to the store that doesn't have a website. The "UNDEFINED" make users misunderstood the code because they would think the website URL is broken. We have trouble trying to change the word "UNDEFINED" to a blank space, but we ended up providing a notice that states that the "UNDEFINED" means there's no website available for the store.
